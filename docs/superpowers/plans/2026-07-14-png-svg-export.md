# PNG/SVG Graph Export Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let users export the rendered graph as a PNG or SVG image from both `export` and `serve` modes, via a toolbar button + `E` keyboard shortcut.

**Architecture:** Purely client-side change to the single shared `htmlTemplate` in `internal/renderer/export.go` (used by both `ExportHTML` and `ServeHTML`). Adds one new bundled JS library (`cytoscape-svg`) following the exact pattern of the four libraries already bundled. No new Go types, no new CLI flags, no server/network changes — export happens via `<a download>` on a blob/data-URI generated by Cytoscape in the browser.

**Tech Stack:** Go `text/template` (existing `htmlTemplate`), vanilla JS, Cytoscape.js core (`cy.png`, already available) + `cytoscape-svg` extension (`cy.svg`, newly bundled), Make + curl for the offline JS bundle.

## Global Constraints

- No new Go dependencies, no new CLI flags, no server-side changes (spec: Scope).
- Export must work in both `export` and `serve` modes — implement once in the shared `htmlTemplate` (spec: Scope).
- PNG: `cy.png({full: true, bg: null, scale: 2})` — full-graph fit, transparent bg, fixed 2x scale, no UI control for scale (spec: Export mechanics, Non-goals).
- SVG: `cy.svg({full: true, bg: null})` via `cytoscape-svg` extension (spec: Export mechanics).
- Filenames are static: `tf-lens-graph.png` / `tf-lens-graph.svg` — no timestamp/plan-name logic (spec: Export mechanics).
- New bundled library follows the existing pattern exactly: Makefile version var + download line + `bundle-check` entry, `loadBundledJS()` required-file list, CDN `<script src>` fallback tag. `bundle.go` needs no change (`//go:embed js` already covers new files) (spec: Bundle changes).
- Tests live in `internal/renderer/export_test.go`, same style as existing `TestExportHTML_*` tests (string-contains assertions on rendered HTML) — no new test file (spec: Testing).

- [ ] **Step 7: Add the export JS functions**

In `internal/renderer/export.go`, immediately before the `// ── Minimap ──` comment block (~line 1571), add:

```go
// ── Image export ─────────────────────────────────────────────────────────
window.toggleExportMenu = function(){
  document.getElementById('export-menu').classList.toggle('show');
};
document.addEventListener('click', function(e){
  var menu = document.getElementById('export-menu');
  var btn = document.getElementById('export-btn');
  if(menu && menu.classList.contains('show') && !menu.contains(e.target) && e.target !== btn){
    menu.classList.remove('show');
  }
});
window.exportImage = function(format){
  if(!cy) return;
  document.getElementById('export-menu').classList.remove('show');
  var a = document.createElement('a');
  if(format === 'svg'){
    var svg = cy.svg({full: true, bg: null});
    a.href = 'data:image/svg+xml;charset=utf-8,' + encodeURIComponent(svg);
  } else {
    a.href = cy.png({full: true, bg: null, scale: 2});
  }
  a.download = 'tf-lens-graph.' + format;
  a.click();
};
```
